* Arquitectura del Sistema

** Stack Tecnologico
- Frontend: Next.js 16 + App Router + TypeScript
- UI: Tailwind CSS + Lucide icons + ReactMarkdown
- AI: Groq API (llama-3.1-8b-instant gratuito, llama-3.3-70b-versatile premium)
- Base de datos: Supabase (PostgreSQL + Auth + RLS)
- Email: Resend (3.000 emails/mes gratis)
- Deploy: Vercel (Hobby plan)
- Bot: Telegram

* Gestion de Incidentes

** Flujo automatico: de señal a incidente

#+BEGIN_SRC plantuml :file assets/incident-pipeline.png :exports results
@startuml
skinparam backgroundColor #0F172A
skinparam defaultFontSize 12

title Pipeline: Señal → Incidente → Accion

start

:OSINT Sensor recolecta señales;\nnote right: Reddit, GDELT, USGS, RSS\nCada 6h

partition "Clasificacion" {
  :Keywords (GDELT/RSS);\nnote right: 0 coste API\n3 categorias principales
  :Groq (Reddit);\nnote right: llama-3.1-8b-instant\nDetecta primera persona
}

partition "Agrupacion (incident-detector)" {
  :Cluster por tipo + pais + fecha;\nnote right: Mismo tipo y ubicacion\nen 48h = mismo incidente
  if (2+ señales?) then (si)
    :Crear/Actualizar incidente;\nnote right: Incluye:\n- Recomendacion automatica\n- Severidad maxima\n- Fuentes combinadas
  else (no)
    :Señal se queda en DB;\nnote right: Puede agruparse mas tarde
    stop
  endif
}

partition "Expiracion automatica" {
  :Expiracion segun tipo;\nnote right: Protest: 12h\nFlight: 24h\nHealth: 168h
}

partition "Nota del analista (manual)" {
  :Admin añade nota experta;\nnote right: Via /admin/incidents\no API REST
}

partition "Visualizacion publica" {
  :/osint muestra incidente;\nnote right: 3 capas:\n1. Recomendacion auto\n2. Nota analista\n3. Valoracion comunidad
}

stop

@enduml
#+END_SRC

** Flujo automatico
1. OSINT sensor recolecta señales de 5 fuentes
2. Clasificacion por keywords (GDELT/RSS) o Groq (Reddit)
3. Detector agrupa 2+ señales del mismo tipo+ubicacion = incidente
4. Se crea incidente con recomendacion automatica
5. Expira automaticamente segun tipo (12h-168h)

* Solucion de Problemas

** Cron tarda mas de 90s

| Sintoma | Causa probable | Solucion |
|---------+----------------+----------|
| Timeout en Fase 1 | MAEC scrape 26 paises | Reducir a 20 paises |
| Timeout en Fase 3 | Groq API lento | Limitar posts a 10 (actual 20) |
| Timeout en Fase 4 | Newsletter batch grande | Procesar 50 por batch |

** Incidentes no se crean

| Verificacion | Comando | Esperado |
|--------------+---------+----------|
| Hay señales en DB | `SELECT count(*) FROM osint_signals WHERE created_at > now() - interval '48h';` | > 0 |
| Tabla incidents existe | `\dt incidents` | Tabla encontrada |
| Detector funciona | Trigger cron manual | `created > 0` en respuesta |

** Newsletter no se envia

| Verificacion | Causa | Solucion |
|--------------+-------+----------|
| RESEND_API_KEY | Variable no configurada | Añadir a .env.local + Vercel |
| Suscriptores confirmados | No hay suscriptores | Verificar tabla newsletter_subscribers |
| Dia correcto | Weekly solo funciona lunes | Probar daily digest en su lugar |
