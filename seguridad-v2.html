⚠ Aviso de limitación de responsabilidad operacional

Viaje con Inteligencia es un ecosistema de conciencia situacional asistido por fuentes OSINT públicas. Los análisis de riesgo generados son síntesis automatizadas de información pública (MAEC, USGS, GDACS, ACLED, WHO y 73+ fuentes más) combinadas con modelos de IA. No constituyen asesoramiento de seguridad profesional, legal, médico ni consular.

Las condiciones en destino pueden cambiar sin previo aviso. Para operaciones de alto riesgo, viajes a zonas en conflicto, misiones humanitarias o cualquier decisión donde la seguridad personal sea crítica, consulta siempre las recomendaciones oficiales del MAEC y un profesional certificado en seguridad de viaje. Este servicio no reemplaza a ninguno de ellos.

01 · Qué es este ecosistema

Un aggregator OSINT-asistido, no una plataforma de inteligencia operacional

Precisión semántica antes que cualquier otra cosa. Viaje con Inteligencia no genera inteligencia primaria mediante recopilación activa. Correlaciona y contextualiza 73+ fuentes OSINT públicas para que viajeros ordinarios accedan a información que normalmente solo está al alcance de servicios de inteligencia corporativos.

Lo que hacemos

Contextualización OSINT-asistida por IA

Correlacionamos fuentes abiertas (MAEC, USGS, GDACS, ACLED, WHO, World Bank, Global Peace Index, Aviation Safety Network…) y aplicamos un índice IRV compuesto de 16 KPIs en 5 dimensiones ponderadas para 110 países. El Chat IA (Groq/LLaMA) genera recomendaciones personalizadas en tiempo real.

Lo que no hacemos

No generamos inteligencia operacional primaria

No realizamos recopilación activa, vigilancia, análisis HUMINT ni accedemos a fuentes clasificadas. Los análisis dependen de la calidad y actualidad de las fuentes públicas integradas. Consulta la lista completa de fuentes y la metodología IRV.

Público objetivo

El espacio entre el MAEC y la inteligencia corporativa

Viajeros independientes, periodistas freelance, trabajadores de ONG, nómadas digitales y equipos pequeños sin acceso a productos de inteligencia de viaje corporativos. El MAEC ofrece avisos genéricos; los productos empresariales cuestan cinco cifras anuales. Existimos en el espacio intermedio.

Transparencia

Proyecto personal de código abierto

Desarrollado por M. Castillo desde España. El repositorio es público en GitHub. No es un producto corporativo. La arquitectura, limitaciones y evolución se documentan abiertamente en /ecosistema.

04 · Seguridad informática

Defensa en capas — respuestas técnicas concretas

Las preguntas correctas son: ¿qué se almacena?, ¿cuánto tiempo?, ¿está cifrado?, ¿qué garantías hay? Aquí están las respuestas — sin marketing.

Activo

Cifrado en tránsito y en reposo

  • TLS 1.3 en toda la cadena: cliente → Nginx → Next.js → Supabase
  • HSTS con preload — no hay fallback a HTTP
  • Supabase cifra volúmenes con AES-256 en reposo
  • Certificados Let's Encrypt con renovación automática
Activo

Row Level Security (RLS) en base de datos

  • Todas las tablas con datos personales tienen RLS activo
  • Políticas: auth.uid() = user_id — aislamiento total por usuario
  • Una API key comprometida no puede acceder a datos de otros usuarios
  • Auditoría de accesos habilitada en Supabase
Activo

Control propio de infraestructura

  • Migración a Hetzner Cloud EU completada en mayo 2026
  • Elimina la superficie de ataque de edge networks compartidas
  • Acceso SSH con clave pública únicamente — sin contraseñas
  • Firewall UFW + reglas Nginx restrictivas por defecto
Activo

Gestión de secretos

  • Todas las API keys en variables de entorno del servidor — nunca en código
  • Cero secretos comiteados en el repositorio GitHub
  • Rotación programada de credenciales
  • Webhook de Stripe validado con firma HMAC
En progreso

Rate limiting en API públicas y B2B

  • Límites por IP y por user_id en rutas /api/*
  • Protección reforzada en /api/webhooks/stripe
  • API B2B (/precio-api) con autenticación por token
  • Registro de intentos de abuso en Supabase
Planificado

Canal de divulgación de vulnerabilidades

  • /.well-known/security.txt con contacto y PGP key
  • SLA de respuesta a reportes: 72 horas
  • Reconocimiento público a investigadores responsables
  • Auditoría de dependencias con Dependabot
05 · Seguridad IA

Los modelos que usamos y cómo los protegemos

Dos integraciones de IA activas con arquitecturas distintas. Ambas stateless. Ninguna comparte contexto entre usuarios.

Groq API · LLaMA

Chat IA en tiempo real

  • Usado para el Chat IA conversacional (/dashboard)
  • Cada conversación es una sesión completamente nueva
  • No existe historial compartido entre usuarios
  • Input sanitizado antes de enviar — longitud, caracteres de control, patrones de inyección
Anthropic Claude API

Análisis de riesgo compuesto IRV

  • Síntesis de fuentes OSINT en informes estructurados
  • Stateless: cada análisis es una llamada API independiente
  • System prompt con rol estricto y negación explícita de redireccionamiento
  • Anthropic no retiene datos de API en su política estándar
Amenaza mitigada

Prompt injection

  • Función sanitizeItinerary() pre-LLM: elimina caracteres de control, limita longitud, detecta patrones de jailbreak
  • Separación estructural: datos de usuario y system prompt en roles API distintos — nunca concatenación de strings
  • Monitorización de outputs fuera del dominio de análisis de viaje
  • Self-hosting elimina vectores de ataque de edge compartido
AI Act · art. 14 y 50

Transparencia y supervisión humana

  • Todos los análisis generados por IA están identificados como tales
  • Mecanismo de feedback (👍/👎 + comentario) en informes generados
  • El sistema no realiza perfilado discriminatorio, manipulación subliminal ni puntuación social
  • Documentación de fuentes, limitaciones y versiones del modelo mantenida públicamente

07 · Marco legal

Normativa aplicable y cumplimiento

Operamos desde España con usuarios en la Unión Europea. Estas son las normas que nos aplican y cómo las implementamos.

RGPD · Reglamento (UE) 2016/679

Protección de datos personales

  • Base jurídica: consentimiento explícito (art. 6.1.a) para tratamiento del itinerario
  • Minimización: solo los datos estrictamente necesarios para el análisis
  • Tus derechos: acceso, rectificación, supresión, portabilidad y oposición — desde tu perfil o en privacidad@viajeinteligencia.com
  • Infraestructura: Hetzner EU + Supabase EU — datos dentro del EEE
  • APIs externas IA: cubiertas por SCCs del RGPD
AI Act · Reglamento (UE) 2024/1689

Uso responsable de inteligencia artificial

  • Clasificación: riesgo limitado según uso actual (conciencia situacional)
  • Transparencia (art. 50): toda interacción IA identificada como tal
  • Supervisión humana (art. 14): feedback activo en informes generados
  • Prohibiciones (art. 5): sin perfilado discriminatorio, manipulación subliminal ni puntuación social
LSSI-CE · Ley 34/2002

Servicios de la Sociedad de la Información

  • Identificación del prestador en /legal
  • Política de cookies conforme a directrices AEPD
  • Disclaimer operacional en todos los informes IA
  • Comunicaciones comerciales identificadas
PCI DSS · Pagos

Seguridad en pagos delegada en Stripe

  • Stripe Checkout hospedado — datos de tarjeta nunca tocan nuestra infraestructura
  • Clasificación SAQ-A: mínima responsabilidad PCI DSS
  • Stripe: proveedor certificado PCI DSS Level 1
  • Webhook validado con firma HMAC Stripe-Signature

09 · Preguntas frecuentes

Lo que los usuarios nos preguntan

¿Se almacena mi itinerario de viaje?
No de forma permanente. Los datos de itinerario se procesan en sesión y se eliminan automáticamente en menos de 24 horas. Si activas el historial en tu perfil, el resumen del informe (no el itinerario en bruto) puede guardarse hasta que lo elimines tú. Tienes control total desde la configuración de tu cuenta.
¿Qué modelos de IA procesan mis datos?
El Chat IA usa la API de Groq (LLaMA). Los análisis de riesgo compuesto pueden usar Anthropic Claude API. Ambas integraciones son completamente stateless — cada sesión es una llamada API nueva sin historial compartido entre usuarios. Ninguna de las dos APIs retiene datos en su política estándar de uso.
¿Dónde están alojados mis datos?
La aplicación corre en servidores Hetzner Cloud (datacenter EU) bajo control propio — Next.js bare metal con PM2 y Nginx. La base de datos está en Supabase (región EU). Ningún dato personal sale del Espacio Económico Europeo salvo las llamadas a APIs de IA (Groq, Anthropic), realizadas bajo Cláusulas Contractuales Tipo RGPD.
¿Puedo confiar en el análisis para decisiones de seguridad críticas?
Para planificación ordinaria, sí — el IRV aporta contextualización que normalmente no tendrías. Para operaciones de alto riesgo, zonas de conflicto o misiones humanitarias: consulta siempre las recomendaciones oficiales del MAEC y profesionales certificados. Somos una herramienta de conciencia situacional, no un sustituto del criterio profesional.
¿Cumple la plataforma con el RGPD?
Sí. Base jurídica: consentimiento explícito (art. 6.1.a). Infraestructura en EU (Hetzner + Supabase EU). Dispones de derechos de acceso, rectificación, supresión y portabilidad ejercibles desde tu perfil o en privacidad@viajeinteligencia.com.
¿Cómo protegéis contra prompt injection?
Tres capas: (1) función sanitizeItinerary() que valida el input antes del LLM; (2) separación estructural estricta entre datos de usuario e instrucciones del sistema usando roles API distintos — nunca concatenación de strings; (3) monitorización de outputs fuera del dominio de análisis de viaje. El self-hosting en Hetzner elimina adicionalmente vectores de ataque presentes en edge networks compartidas.
¿Dónde reporto una vulnerabilidad?
Escríbenos a seguridad@viajeinteligencia.com. Respondemos en máximo 72 horas. Estamos preparando /.well-known/security.txt con clave PGP para comunicaciones cifradas (Q3 2026). Todo reporte responsable recibe reconocimiento explícito.
